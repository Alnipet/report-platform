# Report Platform (Test Assignment)

Платформа для быстрой генерации отчетов. Этот проект является прототипом, демонстрирующим архитектурный подход к созданию масштабируемой и расширяемой системы отчетов (с time-to-market новых отчетов в 1 рабочий день).

## Как запустить

Всё поднимается одной командой через Docker Compose:

```bash
docker-compose up --build
```

**Ожидаемые порты:**
- **Фронтенд (UI)**: [http://localhost:3000](http://localhost:3000)
- **Бэкенд (API)**: [http://localhost:3001](http://localhost:3001)
- **PostgreSQL**: `5432`
- **Redis**: `6379`

### Возможные нюансы запуска
При первом старте бэкенд может выдавать ошибки `ECONNREFUSED` в логи, пока БД `postgres` не инициализируется и не ответит (healthcheck). Это нормальное поведение прототипа: NestJS перезапустится или дождется базы.
Модели Prisma автоматически загружаются (генерация в Dockerfile: `npx prisma generate`).
Миграции Prisma в текущем сетапе запускаются руками для прозрачности.

Чтобы применить миграции к запущенной БД:
```bash
docker exec -it report_api npx prisma db push
```

## Тестирование интерфейса
Зайдите на `http://localhost:3000` после применения `prisma db push`. Вы увидите два доступных отчета:
1. Активность пользователей (Excel)
2. Финансовая сводка (PDF)

Нажмите "Запустить генерацию". Задача отправится в очередь Redis. Вы увидите историю запусков и меняющийся статус: `PENDING` -> `PROCESSING` -> `COMPLETED`, после чего появится кнопка для скачивания сформированного файла.
