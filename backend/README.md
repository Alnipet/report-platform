# Report Platform - Backend (API & Worker)

Основное ядро платформы отчётов, написанное на инфраструктуре NestJS. Представляет собой модульный монолит, который обрабатывает HTTP запросы и выполняет фоновую генерацию файлов.

## Архитектура внутри модуля
- **ReportsController (API)**: Фасад для взаимодействия с клиентами. Регистрирует задачу в очереди вместо того, чтобы сразу генерировать файл.
- **ReportsProcessor (Worker)**: Слушатель очереди BullMQ. В фоне занимается получением нужного билдера отчётов, формированием документа (Excel/PDF) и сохранением его на диск.
- **ReportRegistry**: Модуль-реестр (Registry Pattern), который хранит коллекцию всех доступных генераторов. 

## Стек технологий
- **NestJS** (Фреймворк)
- **Prisma ORM** + **PostgreSQL** (База данных)
- **BullMQ** + **Redis** (Очереди и асинхронность)
- **ExcelJS** / **PDFKit** (Движки генерации документов)
- **TypeScript**

## Запуск вне Docker
Если вы хотите запустить бэкенд локально (без `docker-compose`), убедитесь, что у вас подняты Postgres и Redis.

```bash
npm install
npx prisma generate
npm run start:dev
```
